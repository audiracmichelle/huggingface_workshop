Section 4 — Fine-Tuning and Foundation Models

In this section we compare two approaches to adapting models for new tasks: fine-tuning an existing model on domain-specific data, and using foundation models that generalize without any additional training.

4.1 What is Fine-Tuning?

Fine-tuning means taking a pre-trained model and re-training some of its layers on a new, domain-specific dataset. The model keeps the general knowledge it learned during pre-training and adapts its output to the new task.

How the TCD SegFormer was built

In Section 3 we compared two models on tree cover detection:

NVIDIA SegFormerTCD SegFormer
Starting point Pre-trained on ImageNet, fine-tuned on ADE20K Started from the NVIDIA SegFormer weights
Fine-tuning data ADE20K (150 scene classes, 20K images) Restor TCD (~4,600 aerial images, 2 classes)
What changed — Classifier head replaced (150 → 2 classes), all layers re-trained on tree cover data
Result General scene segmentation Specialized tree cover detection

The key insight: the NVIDIA and TCD models share the same backbone (mit-b0, 3.7M parameters). Fine-tuning only changed what the model looks for, not how it sees.

The fine-tuning recipe

A typical fine-tuning workflow on Hugging Face involves:

  1. Load a pre-trained model with from_pretrained(), replacing the classifier head for the new number of classes (ignore_mismatched_sizes=True)
  2. Preprocess the dataset using the model's processor to convert images and masks to tensors
  3. Configure training with TrainingArguments (learning rate, epochs, batch size, evaluation strategy)
  4. Train with the HuggingFace Trainer, which handles the training loop, gradient computation, checkpointing, and evaluation
  5. Evaluate using metrics like Mean IoU (Intersection over Union) — the standard metric for segmentation

4.2 Foundation Models: SAM (Segment Anything)

SAM (Segment Anything Model) is a foundation model for image segmentation developed by Meta. Unlike SegFormer, which is trained for specific class labels, SAM can segment any object in an image without knowing what it is.

Fine-tuned model (SegFormer)Foundation model (SAM)
Approach Semantic segmentation — assigns a class label to each pixel Promptable segmentation — segments objects given a prompt (point, box, or text)
Training Fine-tuned on a specific dataset with class labels Pre-trained on 11M images with 1B+ masks
Output Per-pixel class labels (e.g., "tree", "background") Binary masks (object vs. background) without class names
Labeling effort Requires a labeled dataset for your domain No labeling needed — works zero-shot
Best for Known categories, domain-specific accuracy Zero-shot segmentation, interactive annotation, unknown objects
Why SAM matters for science SAM represents a shift toward foundation models for segmentation — models that generalize across domains without fine-tuning. This means you can segment objects in new domains (e.g., cells, satellite imagery, field samples) without collecting and labeling a training dataset first. You can even use SAM to create training labels, then fine-tune a smaller model like SegFormer on those labels.

Try it yourself: Grounded SAM

Grounded SAM combines two models to do text-prompted segmentation:

  1. Grounding DINO — takes a text prompt (e.g., "tree") and finds bounding boxes around matching objects
  2. SAM — takes those boxes and produces precise segmentation masks

Load the data and both models:

from datasets import load_dataset
from transformers import pipeline, SamModel, SamProcessor
import torch

dataset = load_dataset("restor/tcd-nc")
image_train = dataset["train"][0]["image"]
image_test  = dataset["test"][15]["image"]

# Step 1: Grounding DINO detects objects from text
detector = pipeline(
    "zero-shot-object-detection",
    model="IDEA-Research/grounding-dino-tiny"
)

# Step 2: SAM segments the detected regions
sam_model = SamModel.from_pretrained("facebook/sam-vit-base")
sam_processor = SamProcessor.from_pretrained("facebook/sam-vit-base")

Detect "tree" regions and segment them:

def grounded_sam(image, text_prompt, threshold=0.3):
    # Detect bounding boxes from text
    detections = detector(image, candidate_labels=[text_prompt], threshold=threshold)
    if not detections:
        print("No detections found.")
        return [], []

    # Convert boxes to SAM format [[x1, y1, x2, y2], ...]
    boxes = [[d["box"]["xmin"], d["box"]["ymin"],
              d["box"]["xmax"], d["box"]["ymax"]] for d in detections]

    # Run SAM on detected boxes
    inputs = sam_processor(image, input_boxes=[boxes], return_tensors="pt")
    with torch.no_grad():
        outputs = sam_model(**inputs)
    masks = sam_processor.image_processor.post_process_masks(
        outputs.pred_masks,
        inputs["original_sizes"],
        inputs["reshaped_input_sizes"]
    )[0].squeeze(1).numpy()

    return detections, masks
# Run on image_train
detections_train, masks_train = grounded_sam(image_train, "tree")
print(f"image_train: {len(detections_train)} tree regions detected")
# Run on image_test
detections_test, masks_test = grounded_sam(image_test, "tree")
print(f"image_test: {len(detections_test)} tree regions detected")

Visualize the results:

import matplotlib.pyplot as plt
import numpy as np

def show_grounded_sam(image, detections, masks, title):
    fig, ax = plt.subplots(1, 2, figsize=(10, 5))
    ax[0].imshow(image)
    ax[0].set_title(f"Original ({title})")
    ax[0].axis("off")

    ax[1].imshow(image)
    # Overlay all detected tree masks
    combined = np.zeros(masks[0].shape, dtype=bool) if len(masks) > 0 else np.zeros((1,1))
    for m in masks:
        combined = combined | (m > 0.5)
    ax[1].imshow(combined, alpha=0.5, cmap="Greens")
    ax[1].set_title(f"Grounded SAM — 'tree' ({len(detections)} regions)")
    ax[1].axis("off")

    plt.tight_layout()
    plt.show()

show_grounded_sam(image_train, detections_train, masks_train, "image_train")
show_grounded_sam(image_test, detections_test, masks_test, "image_test")
